# 点云轮廓基线估计方法的问题发现与改进

## 1. 实验目的

本实验针对轮胎点云展开后的轮廓基线估计问题，目标是从原始三维点云中构造规则二维高度图，估计轮胎表面的大尺度背景起伏，并通过原始高度减去基线得到残差图，从而突出轮胎表面的字符、纹理、缺陷和局部凸起。

本实验不是单纯运行老师给出的程序，而是按照“初始方法复现 - 问题发现 - 原因分析 - 方法改进 - 对比验证”的流程，对现有基线估计算法进行改进。

## 2. 初始方法

给定点云 `pc1.mat` 中的 `pc1.Location`，取三列坐标分别为：

- `X`：轮胎展开后的横向坐标；
- `Y`：扫描步进方向坐标；
- `Z`：高度或轮廓值。

首先去除无效点和全零点，然后将点云映射到规则二维网格，得到高度图 `Zmap(y,x)`。其中 `Y` 方向按照 `scanStep = 0.1` 量化，`X` 方向按照 `xGridStep = 0.2` 分箱，同一网格内多个点取平均值，少量空洞使用最近邻补全。

初始方法采用两步一维基线估计：

1. 对每条扫描线，即 `Zmap` 的每一行，沿 `X` 方向做一维基线估计，得到第一遍基线 `B1`。
2. 计算第一遍残差 `R1 = Zmap - B1`。
3. 对 `R1` 的每一列，沿 `Y` 方向估计残余低频趋势，得到第二遍基线 `B2`。
4. 合成总基线：

```text
baselineTotal = B1 + B2
detailMap = Zmap - baselineTotal
```

一维基线估计的基本做法是：将一维信号按固定宽度划分为不重叠窗口，在每个窗口内取 10% 低分位值作为背景控制点，再用 `pchip` 插值得到连续基线。

本次脚本中初始复现版本参数为：

| 参数 | 数值 |
|---|---:|
| `scanStep` | 0.1 |
| `xGridStep` | 0.2 |
| `blockWidthX` | 5.0 |
| `blockWidthY` | 1.0 |
| `percentile` | 10 |
| `minPtsPerBlockX` | 3 |
| `minPtsPerBlockY` | 2 |

运行 Python 脚本后，初始方法结果保存在：

## 3. 初始结果中的问题

### 3.1 字符局部被削弱

在不满意结果中，部分字符笔画亮度不足。例如数字或字母的某些边缘没有像相邻笔画一样清晰高亮，说明该局部结构在基线扣除过程中被错误当作背景的一部分。

原因是固定窗口的尺度可能与字符笔画尺度接近。当窗口刚好覆盖字符局部结构时，窗口内的低分位统计值不再能稳定代表真实背景，插值得到的基线会偏离合理下包络，最终导致字符残差被削弱。

### 3.2 竖向拖尾伪影

不满意结果中可以看到高亮点附近出现细长竖向尾巴，尤其在点状凸起下方更明显。这类结构并不是轮胎表面的真实纹理，而是展开和基线估计带来的伪影。

原因主要包括：局部异常控制点会通过 `pchip` 插值影响邻近区域；第一遍残差中的异常结构在第二遍沿 `Y` 方向处理时可能继续传播；不稳定的边界或稀疏点也会使残差沿扫描方向形成拖尾。

### 3.3 暗网格和块边界

不重叠分块使每个窗口独立产生一个低值控制点，相邻窗口之间的代表值可能跳变。虽然 `pchip` 可以保证曲线连续，但不能完全消除块级统计造成的局部趋势突变。二维两步处理后，横向和纵向块边界叠加，就可能形成网格状暗纹或块状感。

### 3.4 边界厚化

展开图上下边界附近容易出现较厚亮带或黑带。这是因为边界缺少足够邻域，端点处只能使用端值约束或最近邻填充，基线估计稳定性差。

## 4. 改进方法

本实验采用“滑动窗口 + 低分位值 + 控制点轻微平滑”的改进方法。

与初始方法相比，改进点如下：

1. 保留低分位值统计，而不是使用最小值，降低极端低点对基线的影响。
2. 用滑动窗口替代不重叠分块，使相邻窗口高度重叠，控制点更密集、更连续。
3. 对控制点序列做轻微移动中值平滑，减弱孤立异常控制点造成的局部过冲。
4. 保留“先逐扫描线沿 X 方向处理，再沿 Y 方向修正”的顺序，避免先跨扫描线混合不同字符结构。

改进版参数为：

| 参数 | 数值 |
|---|---:|
| `windowWidthX` | 5.0 |
| `windowStepX` | 1.0 |
| `windowWidthY` | 1.0 |
| `windowStepY` | 0.2 |
| `percentile` | 10 |
| `smoothCtlX` | 3 |
| `smoothCtlY` | 3 |

## 5. 参数敏感性分析

为了分析低分位值参数对结果的影响，脚本额外运行了 `q = 5, 10, 15` 三组滑动窗口实验，结果保存在：

```text
results/exp4_baseline_estimation_python/parameter_sensitivity/
```

一般来说：

- `q = 5` 更接近下包络，细节保留较多，但仍可能受异常低点影响；
- `q = 10` 通常在细节保留和背景平滑之间较平衡；
- `q = 15` 对异常点更鲁棒，但基线可能更高，细小字符存在被过度扣除的风险。

脚本同时输出半定量指标：

- `bgStd`：背景区域标准差，越小说明背景越平坦；
- `brightContrast`：高亮细节与背景的对比；
- `negativeDip`：负向凹陷程度；
- `edgeAbsMean`：边界区域异常强度；
- `verticalTailProxy`：沿 `Y` 方向的残差变化强度，可作为拖尾伪影的粗略参考。

这些指标不等价于真值评价，但可以辅助说明不同参数下背景平坦性、边界稳定性和伪影强弱的变化。

## 6. 实验结论

初始的“不重叠分块 + 低分位值 + pchip”方法能够较好地去除轮胎表面的大尺度起伏，并在残差图中显示出字符、标记点和局部纹理。但由于窗口不重叠、控制点稀疏以及边界约束不足，结果中会出现字符局部被削弱、竖向拖尾、暗网格和边界厚化等问题。

改进后的滑动窗口方法增加了控制点密度，使相邻位置的基线估计更加连续。结合低分位值和控制点轻微平滑后，块边界感和局部拖尾有所减弱，背景更加均匀，同时主要字符和轮胎标识仍能保留。

该方法仍有局限：当字符尺度与窗口尺度接近时，仍可能出现局部误扣除；边界区域由于点云稀疏和邻域不足，仍需要裁剪、镜像扩展或更强的二维曲面约束进一步改善。后续可以尝试真正的二维鲁棒曲面拟合，或在字符候选区域加入保护掩膜，避免显著局部结构参与基线估计。

## 7. 运行方式

本实验已改为 Python 实现。先创建虚拟环境并安装依赖：

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
```

然后运行：

```bash
.venv/bin/python code/baseline_estimation_python.py
```

脚本默认只截取 `Y in [-200, 0]` 的局部区域，适合快速完成实验对比。如果需要生成完整轮胎展开范围结果，运行：

```bash
.venv/bin/python code/baseline_estimation_python.py --full
```

也可以快速测试一小段数据：

```bash
.venv/bin/python code/baseline_estimation_python.py --y-range -20 0 --no-parameter-sensitivity --out results/python_smoke_test
```
